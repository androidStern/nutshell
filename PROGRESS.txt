What v0.1.23 is (goal: ../setup-onboarding-and-feedback-loops-goal.md, approved 2026-06-10):
- Setup verifies every source with its real probe through the Nutshell.app identity (hidden __probe bridge); ready requires a passing probe; probe/retry/skip loop; permission step BEFORE verification; re-run opens with a probed status table and fixes only what fails; real bounded smoke sync (fictional "handed off" wording removed); already-imported archives detected from store-derived backfill status.
- Every problem finding carries guidance {state, fix, confirm, url?} authored in per-plugin catalogs (TracePlugin.findings) + system/setup catalogs; universal invariant test renders every emittable code through every surface and bans raw finding construction outside catalog modules. Catch-all auth codes split one-code-one-state (keychain-blocked = needs_permission, never needs_auth).
- Scheduler self-heals: degraded source gets one bounded probe per scheduled run; pass flips ready + syncs; fail refreshes one deduped finding; *_rate_limited codes back off (catalog naming contract). Sync failing on needs_auth/needs_permission re-degrades the source.
- Doctor: root-cause-first text (FDA/app criticals lead, needs_permission symptoms collapse), aliases x/notes/podcast, no-arg works; sync prints human text with skipped-source fixes; health carries standing backfill-pending line with exact import command; dashboard renders fix text (guidance persisted in store via guidance_json column + migration v2).
- local_os sources (Notes/Podcasts) get REAL probes when health/doctor run inside the app identity (NUTSHELL_BUNDLE env check) — previously never probed.
